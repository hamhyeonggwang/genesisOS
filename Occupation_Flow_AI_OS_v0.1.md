# Occupation Flow AI OS

> Version: 0.1 (Foundation) Authoring philosophy: Workflow First · AI
> Second

------------------------------------------------------------------------

# Vision

AI는 목적이 아니라 **작업(Occupation)의 흐름을 연결하는 운영체제**이다.

최종 목표는 Claude를 잘 사용하는 것이 아니라, **Occupation Flow AI
OS**를 구축하여 임상, 연구, 교육, 개발을 하나의 AI Workflow로 연결하는
것이다.

# Eight Layers

## L1 Foundation

-   Claude Desktop
-   Claude Code
-   Terminal
-   Git / GitHub
-   Project Management

## L2 Memory

-   CLAUDE.md
-   프로젝트 규칙
-   작업치료 철학
-   ICF / PEO / EBP
-   개발 원칙

## L3 Commands

반복 작업을 명령으로 추상화한다.

예) - /report - /paper - /lecture - /gas - /icf - /deploy

## L4 Skills

전문성을 재사용 가능한 Skill로 만든다.

### Clinical

-   ICF Mapper
-   OT Report Writer
-   Intervention Designer

### Research

-   Research Analyzer
-   Evidence Reviewer
-   APA Writer

### Education

-   Lecture Planner
-   Quiz Generator
-   Slide Builder

### Development

-   GAS Architect
-   n8n Builder
-   Next.js Builder
-   UX Reviewer

## L5 Knowledge OS

모든 지식을 Markdown 기반으로 연결한다.

Sources → Markdown → Obsidian → Knowledge Graph

## L6 Connected AI

MCP를 통해 외부 도구를 연결한다.

-   GitHub
-   Drive
-   Calendar
-   Figma
-   Supabase
-   n8n

## L7 AI Team

AI를 역할별 전문가 팀으로 운영한다.

-   PM
-   Researcher
-   Clinical OT
-   Full Stack Developer
-   UX Designer
-   Reviewer

## L8 Automation

반복 업무를 자동 실행한다.

Hooks → Headless → CI/CD → n8n → Scheduled Routines

# Four Operating Domains

## Clinical

Evaluation → ICF → Goal → Intervention → Outcome

## Research

Question → Search → Critical Appraisal → Summary → Knowledge

## Education

Evidence → Learning Objectives → Lecture → Practice → Assessment

## Development

Idea → PRD → Design → Build → Test → Deploy

------------------------------------------------------------------------

# Principles

1.  Workflow First
2.  AI Second
3.  Knowledge becomes Assets
4.  Every repeated task becomes a Command
5.  Every reusable workflow becomes a Skill
6.  Every project starts with CLAUDE.md
7.  Markdown is the single source of truth.
8.  Build once. Reuse everywhere.
